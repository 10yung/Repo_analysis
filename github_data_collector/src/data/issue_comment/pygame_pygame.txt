On Ubuntu 18.04 using Python 3.6.8. If I write a code that assigns
Button <kbd>F</kbd> to redrawing with `pygame.display.set_mode( ... , FULLSCREEN)`
Button <kbd>G</kbd> to toggling fullscreen with the method `toggle_fullscreen()`
Button <kbd>R</kbd> to drawing alternatively a completely blue or completely red surface (just as an example).

This is what I observe
1. Using pygame 2.0.0.dev6, when going fullscreen with either <kbd>F</kbd> or <kbd>G</kbd> the screen does the following:
    1. flashes a few times
    2. goes in the task bar as a minimized icon
    3. if I click on the icon the screen flashes a few more times and finally we are fullscreen
    4. the screen is entirely black and the button <kbd>R</kbd> does *not* flip the colors
2. Still using pygame 2.0.0.dev6. In this case the <kbd>G</kbd> and the <kbd>F</kbd> button behave differently: when going back from fullscreen to windowed with <kbd>G</kbd>, the <kbd>R</kbd> button doesn't flip the colors, even in the windowed version. When using <kbd>F</kbd> instead it works.
3. With pygame version 2.0.0.dev3 the <kbd>G</kbd> button does not work at all, while <kbd>F</kbd> has the same behavior as before.
4. If I use `FULLSCREEN|SCALED` in place of `FULLSCREEN` in the method for <kbd>F</kbd> the problems 1.1., 1.2 and 1.3. are gone: the app goes to fullscreen immediately. But the problem 1.4. persists and furthermore the program does not accept inputs anymore.
