<details>
<summary>Changelog</summary>
<p><em>Sourced from <a href="https://github.com/terser/terser/blob/master/CHANGELOG.md">terser's changelog</a>.</em></p>
<blockquote>
<h2>v4.6.2</h2>
<ul>
<li>A bug where functions were inlined into other functions with scope conflicts has been fixed.</li>
<li><code>/*#__NOINLINE__*/</code> annotation fixed for more use cases where inlining happens.</li>
</ul>
<h2>v4.6.1</h2>
<ul>
<li>Fixed an issue where a class is duplicated by reduce_vars when there's a recursive reference to the class.</li>
</ul>
<h2>v4.6.0</h2>
<ul>
<li>Fixed issues with recursive class references.</li>
<li>BigInt evaluation has been prevented, stopping Terser from evaluating BigInts like it would do regular numbers.</li>
<li>Class property support has been added</li>
</ul>
<h2>v4.5.1</h2>
<p>(hotfix release)</p>
<ul>
<li>Fixed issue where <code>() =&gt; ({})[something]</code> was not parenthesised correctly.</li>
</ul>
<h2>v4.5.0</h2>
<ul>
<li>Inlining has been improved</li>
<li>An issue where keep_fnames combined with functions declared through variables was causing name shadowing has been fixed</li>
<li>You can now set the ES version through their year</li>
<li>The output option <code>keep_numbers</code> has been added, which prevents Terser from turning <code>1000</code> into <code>1e3</code> and such</li>
<li>Internal small optimisations and refactors</li>
</ul>
<h2>v4.4.3</h2>
<ul>
<li>Number and BigInt parsing has been fixed</li>
<li><code>/*#__INLINE__*/</code> annotation fixed for arrow functions with non-block bodies.</li>
<li>Functional tests have been added, using <a href="https://github.com/terser/terser-functional-tests">this repository</a>.</li>
<li>A memory leak, where the entire AST lives on after compression, has been plugged.</li>
</ul>
<h2>v4.4.2</h2>
<ul>
<li>Fixed a problem with inlining identity functions</li>
</ul>
<h2>v4.4.1</h2>
<p><em>note:</em> This introduced a feature, therefore it should have been a minor release.</p>
<ul>
<li>Fixed a crash when <code>unsafe</code> was enabled.</li>
<li>An issue has been fixed where <code>let</code> statements might be collapsed out of their scope.</li>
<li>Some error messages have been improved by adding quotes around variable names.</li>
</ul>

<h2>Unauthorized File Access</h2>
<p>Versions of the npm CLI prior to 6.13.3 are vulnerable to an Arbitrary File Write. It is possible for packages to create symlinks to files outside of the<code>node_modules</code> folder through the <code>bin</code> field upon installation. A properly constructed entry in the package.json bin field would allow a package publisher to create a symlink pointing to arbitrary files on a userâ€™s system when the package is installed.</p>
<p>This behavior is possible through install scripts. This vulnerability bypasses a user using the --ignore-scripts install option.</p>
<p>Thank you Daniel Ruf for responsibly reporting the issue!</p>
<p>Further information: <a href="https://blog.npmjs.org/post/189618601100/binary-planting-with-the-npm-cli">npm blog post</a></p>
<p>Affected versions: &lt; 6.13.3</p>
</blockquote>

<blockquote>
<p><strong>Low severity vulnerability that affects npm</strong></p>
<h2>Arbitrary File Write</h2>
<p>Versions of the npm CLI prior to 6.13.3 are vulnerable to an Arbitrary File Write. It fails to prevent access to folders outside of the intended <code>node_modules</code> folder through the <code>bin</code> field. A properly constructed entry in the package.json <code>bin</code> field would allow a package publisher to modify and/or gain access to arbitrary files on a userâ€™s system when the package is installed.</p>
<p>This behavior is possible through install scripts. This vulnerability bypasses a user using the <code>--ignore-scripts</code> install option.</p>
<p>Thank you Daniel Ruf for reporting the issue!</p>
<p>Further information: <a href="https://blog.npmjs.org/post/189618601100/binary-planting-with-the-npm-cli">npm blog post</a></p>
<p>Affected versions: &lt; 6.13.3</p>
</blockquote>
<p><em>Sourced from The GitHub Security Advisory Database.</em></p>
<blockquote>
<p><strong>Low severity vulnerability that affects npm</strong></p>
<h2>Arbitrary File Overwrite</h2>
<p>Versions of  the npm CLI prior to 6.13.4 are vulnerable to an Arbitrary File Overwrite. It fails to prevent existing globally-installed binaries to be overwritten by other package installations. For example, if a package was installed globally and created a <code>serve</code> binary, any subsequent installs of packages that also create a <code>serve</code> binary would overwrite the previous <code>serve</code> binary.</p>
<p>This behavior is still allowed in local installations and also through install scripts. This vulnerability bypasses a user using the --ignore-scripts install option.</p>
<p>Thank you to Daniel Ruf for reporting the issue!</p>
<p>Further information: <a href="https://blog.npmjs.org/post/189618601100/binary-planting-with-the-npm-cli">npm blog post</a></p>
<p>Affected versions: &lt; 6.13.4</p>
