## Describe the bug

<details><summary>Long Description</summary>
<p>
My test is running on BrowserStack. A call to `waitForElementNotPresent` inside a `perform` call took longer to return from BrowserStack than the `perform` timeout allowed (`perform` timeout is 15000ms, BrowserStack sent a response after 24000ms). `perform` call failed, as one would expect, and Nightwatch began to execute my global `afterEach` function (source of which can be found [here](https://gist.github.com/MicahLC/dfc55e8905b997cab0e12f16202f69f8#file-globalaftereach-js)). In my verbose log, the beginning of the `afterEach` function is on [line 644](https://gist.github.com/MicahLC/dfc55e8905b997cab0e12f16202f69f8#file-timing_fail_log-txt-L644).

As part of the `afterEach` call when a test fails, I make several calls to the browser for the page source, the browser logs, our redux store, and the final URL. [At line 669 of the verbose log](https://gist.github.com/MicahLC/dfc55e8905b997cab0e12f16202f69f8#file-timing_fail_log-txt-L669), you can see the call for the page source. You can also see that the next line is the return of the `waitForElementNotPresent` call from BrowserStack, followed by the response with the page source.

The rest of my calls to the browser to get debugging data appear to go through, and then on [line 721](https://gist.github.com/MicahLC/dfc55e8905b997cab0e12f16202f69f8#file-timing_fail_log-txt-L721), the `end` call begins, including capturing the screenshot and deleting the session. But on [line 739](https://gist.github.com/MicahLC/dfc55e8905b997cab0e12f16202f69f8#file-timing_fail_log-txt-L739), the `afterEach` call times out (mostly because it was delayed by BrowserStack), and then `end` is called again, but the screenshot and `delete` calls fail, and then the test isn't retried.

Nightwatch also didn't output the XML for the test suite/file, but the command failed on the CLI, so the Jenkins job failed with no test failures reported, which is very undesirable.
</details>
