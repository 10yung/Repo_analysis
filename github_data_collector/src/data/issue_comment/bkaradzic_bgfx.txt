Original bgfx has some combo box control to dynamically change renderer type. However, it is disabled by default. Also the original bgfx implementation is not complete and never work. It actually does not change the renderer type. When user selects a different renderer type actually the same renderer is used. This is because it was intended (I think) to restart the current application object (AppI&ast; object) with new s_restartArgs command line parameters. However, s_restartArgs was never applied due incomplete implementation. Also s_restartArgs were not specified in the original version (except "app restart").
So, that fork contains the required changes (ad-hoc changes) to correctly pass s_restartArgs to the new instance of AppI&ast; object. Also, it specifies the correct command line arguments for s_restartArgs for restarting to use a new renderer type, for instance, use "--gl" command line argument for OpenGL renderer type, "--d3d11" for Direct3D 11 and so on.
