On a system with multiple .net versions, one being older than 4.7.2, but the other being the latest 4.8, HPOneView.500 does not load and complains it is missing the required .NetFramework 4.7.2 client.

I checked the code in HPOneView.500.psm1, and it seems it is expecting only one value being returned from this command in line 33:

Get-ChildItem 'HKLM:\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full\' |  Get-ItemPropertyValue -Name Release

However, the system where it is throwing the error, returns two values:

461308 (older version)
528049 (4.8)

According to the MS documentation, Get-ChildItem will display subkeys from the specified registry key, not the key itself, see

https://docs.microsoft.com/en-us/powershell/module/microsoft.powershell.core/about/about_registry_provider?view=powershell-6

"Each registry key can also have subkeys. When you use Get-Item on a registry key, the subkeys are not displayed. The Get-ChildItem cmdlet will show you children items of the "Spooler" key, including each subkey's properties. The parent keys properties are not shown when using Get-ChildItem."

So I think the script should be using Get-Item instead of Get-ChildItem, or continue to use Get-ChildItem but then it should be able to handle multiple numbers.
